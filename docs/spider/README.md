# Spider Text-to-SQL with LLM-Judge Labels

This dataset extends [Spider 1.0](https://yale-lily.github.io/spider) with SQL predictions from `gpt-5.4-mini` and two correctness labels per example: a hybrid ground truth label and an LLM judge label from `gpt-5.4`.

## Files

| File | Description |
|------|-------------|
| `spider_dataset.parquet` | Full dataset with predictions and labels |
| `scripts/` | Reproduction scripts (see below) |

## Dataset statistics

- **Source**: Spider 1.0 training split (`train_spider.json`)
- **Databases**: the 10 Spider databases with the most examples (default), selected by running `scripts/explore_dataset.py`
- **Labels**: binary (0 = incorrect, 1 = correct)

## Schema

| Column | Type | Description |
|--------|------|-------------|
| `example_id` | string | Unique identifier (`spider_train_{original_index}`) |
| `db_id` | string | Database name |
| `question` | string | Natural language question |
| `gold_sql` | string | Reference SQL query from the Spider benchmark |
| `predicted_sql` | string | SQL generated by `gpt-5.4-mini` |
| `llm_judge_label` | int | 1 if `gpt-5.4` judges the prediction correct, 0 otherwise |
| `ground_truth_label` | int | 1 if the predicted SQL correctly answers the question, 0 otherwise |
| `llm_judge_reasoning` | string | Free-text rationale from the LLM judge for its label |
| `ground_truth_reasoning` | string | Free-text rationale from the ground-truth annotator for its label (only present when the LLM fallback is triggered) |

Ground truth labels are generated in a hybrid way: both the gold and predicted queries are executed against the database, and the label is 1 if their outputs match. When execution fails or the outputs differ, a `gpt-5.4` call compares the two queries using the full database schema (tables, columns, types, primary keys, foreign keys) to determine correctness. The LLM judge prompt also includes the full database schema.

## Reproduction

Install dependencies:

```bash
pip install -r scripts/requirements.txt
```

Then run the scripts in order from the repository root.

**Step 1 — Download and extract Spider 1.0:**

```bash
python scripts/download_dataset.py
```

By default the script downloads from a hardcoded Google Drive file ID. If that link breaks, pass a direct URL or a local archive instead:

```bash
# Use a direct download URL
python scripts/download_dataset.py --url https://example.com/spider.zip

# Use a locally downloaded archive
python scripts/download_dataset.py --archive-path /path/to/spider.zip

# Override the Google Drive file ID or output directory
python scripts/download_dataset.py --gdrive-file-id <new-id> --output-dir data/spider
```

On success, the output directory (`data/spider/` by default) will contain:

```
data/spider/
├── train_spider.json       # 7,000 training questions with gold SQL queries
├── tables.json             # Schema definitions for all 200 databases
                            # (tables, columns, types, primary/foreign keys)
└── database/
    ├── academic/
    │   ├── academic.sqlite
    │   └── schema.sql
    ├── activity_1/
    │   ├── activity_1.sqlite
    │   └── schema.sql
    └── ...                 # 200 databases total, one directory each
```

**Step 2 — Inspect per-database statistics to inform database selection:**

```bash
python scripts/explore_dataset.py
```

By default the script reads from `data/spider/train_spider.json`. Pass `--data-path` to use a different file:

```bash
python scripts/explore_dataset.py --data-path /path/to/train_spider.json
```

**Step 3 — Generate SQL predictions:**

Accepts any `--model` from OpenAI. Run with `--help` for the full list of options (scope, retries, checkpointing). `OPENAI_API_KEY` must be set.

```bash
python scripts/generate_predictions.py --model gpt-5.4-mini
```

The script defaults to `--n-datasets 10` (the 10 databases with the most examples).

The script prints the output path on completion:

```
Done. 1234 records written to /abs/path/to/data/predictions_gpt-5.4-mini.jsonl
```

Note that path — you will pass it as `--input` in the next step.

**Step 4 — Generate ground truth labels:**

```bash
python scripts/generate_ground_truths.py --input <predictions path from step 3>
```

The script prints its output path on completion. Note that path — you will pass it to `build_dataset.py` in step 6.

**Step 5 — Generate LLM judge labels:**

Pass the predictions file from step 3 as `--input`. Accepts any `--model` from OpenAI. `OPENAI_API_KEY` must be set.

```bash
python scripts/generate_llm_judge_labels.py --model gpt-5.4 --input <predictions path from step 3>
```

The script prints its output path on completion. Note that path — you will pass it to `build_dataset.py` in step 6.

Steps 3, 4, and 5 write checkpointed JSONL files under `data/` and can be safely interrupted and resumed.

**Step 6 — Merge and push to HuggingFace:**

Pass the three output paths noted in steps 3, 4, and 5 as `--predictions`, `--ground-truths`, and `--judge-labels`.

```bash
python scripts/build_dataset.py \
    --predictions <predictions path from step 3> \
    --ground-truths <ground_truths path from step 4> \
    --judge-labels <llm_judge_labels path from step 5> \
    --hf-repo glide-py/spider-text-to-sql
```
